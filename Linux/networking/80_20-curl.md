# 🧠 80/20 Guide to `curl`

## 1) Core Mental Model

At its simplest, `curl` = **make an HTTP request from the terminal**

```bash
curl https://api.example.com
```

Defaults:

* Method: `GET`
* Output: response body → stdout

---

## 2) The 5 Things You Actually Do Most

### ① GET requests (fetch data)

```bash
curl https://api.example.com/users
```

Add query params:

```bash
curl "https://api.example.com/users?page=2&limit=10"
```

---

### ② POST requests (send data)

```bash
curl -X POST https://api.example.com/users \
  -d "name=John&age=30"
```

JSON (very common):

```bash
curl -X POST https://api.example.com/users \
  -H "Content-Type: application/json" \
  -d '{"name":"John","age":30}'
```

---

### ③ Add headers (auth, content type, etc.)

```bash
curl -H "Authorization: Bearer TOKEN" \
     -H "Accept: application/json" \
     https://api.example.com
```

---

### ④ Save output to a file

```bash
curl -o file.json https://api.example.com/data
```

Auto filename:

```bash
curl -O https://example.com/file.zip
```

---

### ⑤ Debug / inspect request

Show headers:

```bash
curl -i https://api.example.com
```

Verbose (full request/response):

```bash
curl -v https://api.example.com
```

---

## 3) The “Power Trio” You’ll Use Constantly

These 3 flags handle most API work:

* `-H` → headers
* `-d` → body data
* `-X` → HTTP method

Example:

```bash
curl -X PUT https://api.example.com/users/1 \
  -H "Authorization: Bearer TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"name":"Alice"}'
```

---

## 4) Authentication Patterns

### Bearer token

```bash
-H "Authorization: Bearer TOKEN"
```

### Basic auth

```bash
-u username:password
```

---

## 5) Handling JSON Cleanly

Pretty-print with `jq`:

```bash
curl https://api.example.com | jq
```

---

## 6) Common Pitfalls (worth knowing early)

* `-d` **implies POST** (even if you don’t specify `-X POST`)
* Missing quotes around JSON → breaks requests
* Headers are case-insensitive but formatting matters
* Always check with `-v` when debugging
