`nmap` looks massive, but its practical core is tight: **discover hosts, scan ports, fingerprint services**. The 80/20 is about asking *what’s there, what’s open, and what is it*—quickly.

---

# The 80/20 Guide to Nmap

## Mental Model (core idea)

`nmap` = **active probing of network surfaces**

It:

* sends crafted packets
* observes responses
* infers:

  * open ports
  * running services
  * OS / versions (optionally)

Think: **reconnaissance engine for networks**

---

## The One Command to Remember

```bash id="h1j7ke"
nmap -sC -sV target
```

Example:

```bash id="z8c9fh"
nmap -sC -sV 192.168.1.10
```

### What this gives you:

* `-sC` → default scripts (safe, useful checks)
* `-sV` → service/version detection

This covers most real-world scans.

---

## The 3 Core Use Cases

## 1. “What ports are open?”

```bash id="rqgqzj"
nmap target
```

Output:

```id="p6a1ve"
PORT   STATE SERVICE
22/tcp open  ssh
80/tcp open  http
```

---

## 2. “What services are running?”

```bash id="j7k5i0"
nmap -sV target
```

Example:

```id="9qgqmv"
80/tcp open  http nginx 1.18.0
```

---

## 3. “Give me a useful default scan”

```bash id="hwb5qa"
nmap -sC -sV target
```

This is your baseline reconnaissance command.

---

## The 3 Flags That Matter Most

### 1. `-p` (port selection)

```bash id="3mx1o9"
nmap -p 22,80,443 target
```

Or full scan:

```bash id="tb6o2q"
nmap -p- target
```

---

### 2. `-sV` (service detection)

```bash id="v6z8wr"
nmap -sV target
```

Identifies versions (e.g., Apache 2.4.41).

---

### 3. `-sC` (default scripts)

```bash id="w7g4b1"
nmap -sC target
```

Runs safe scripts:

* SSL info
* HTTP titles
* basic vuln checks

---

## High-Value Patterns

### Scan a specific host quickly

```bash id="qg1f1r"
nmap 192.168.1.1
```

---

### Scan all ports (important)

```bash id="i3g7fh"
nmap -p- target
```

Default only scans ~1000 ports—this finds hidden services.

---

### Combine everything (most useful real-world)

```bash id="y8u7wd"
nmap -p- -sC -sV target
```

---

### Scan a subnet

```bash id="4j3r8r"
nmap 192.168.1.0/24
```

---

### Fast scan (less thorough)

```bash id="5p7x3v"
nmap -F target
```

---

## Understanding Output (minimal decoding)

Columns:

* **PORT** → port number
* **STATE** → open / closed / filtered
* **SERVICE** → guessed service

---

## The Only States You Need

* `open` → service is accepting connections
* `closed` → reachable, but nothing listening
* `filtered` → blocked (firewall)

---

## Performance Insight

Default scan:

* ~1000 common ports

Full scan:

```bash id="2k9t2m"
nmap -p-
```

→ 65,535 ports

Trade-off:

* speed vs completeness

---

## Privileged Scans (important nuance)

```bash id="p3n9px"
sudo nmap -sS target
```

* `-sS` → SYN scan (stealthier, faster)
* requires root

Without sudo:

* falls back to slower connect scan

---

## Service Enumeration (light touch)

```bash id="x4k9pa"
nmap -sV --version-intensity 5 target
```

More aggressive detection = more accurate, slower.

---

## Pitfalls to Avoid

### 1. Only scanning default ports

You’ll miss services.

Use:

```bash id="r2g7jw"
nmap -p-
```

---

### 2. Misinterpreting “filtered”

It usually means:

* firewall
* packet drop

Not necessarily “secure,” just “invisible.”

---

### 3. Running loud scans on networks you don’t control

`nmap` is detectable and sometimes legally sensitive.

---

## Minimal Cheat Sheet

```bash id="n9v3km"
# basic scan
nmap target

# service detection
nmap -sV target

# default scripts + versions
nmap -sC -sV target

# full port scan
nmap -p- target

# full + scripts + versions
nmap -p- -sC -sV target

# subnet scan
nmap 192.168.1.0/24

# fast scan
nmap -F target
```

---

## Final Takeaway

You don’t need most of `nmap`.

If you internalize:

* `nmap target`
* `-p-` (all ports)
* `-sV` (what is it?)
* `-sC` (quick intelligence)

…you can do meaningful network reconnaissance immediately.
