Here’s a tight, high-signal **80/20 guide to cron on Ubuntu**—just the pieces that give you most of the practical leverage as a sysadmin.

---

## 1) Core Mental Model

* Cron = time-based job scheduler driven by the **daemon**: cron
* Config lives in:

  * User crontab: `crontab -e`
  * System-wide: `/etc/crontab`, `/etc/cron.*/`
* Each line = **schedule + command**

---

## 2) Cron Syntax (you only need this)

```
* * * * * command
│ │ │ │ │
│ │ │ │ └─ Day of week (0–7, Sun=0/7)
│ │ │ └── Month (1–12)
│ │ └──── Day of month (1–31)
│ └────── Hour (0–23)
└──────── Minute (0–59)
```

### High-value patterns

```
*/5 * * * *      → every 5 minutes
0 * * * *        → every hour
0 2 * * *        → daily at 02:00
0 2 * * 0        → Sundays at 02:00
0 2 1 * *        → 1st of every month
@reboot          → at system boot
```

---

## 3) Editing & Managing

```bash
crontab -e      # edit your jobs
crontab -l      # list jobs
crontab -r      # remove all jobs
```

System jobs:

```bash
sudo nano /etc/crontab
ls /etc/cron.daily/
```

---

## 4) Environment Gotchas (this is where most people fail)

Cron runs in a **minimal shell**:

* No full `$PATH`
* No interactive shell config (`.bashrc` ignored)

### Fix it explicitly:

```bash
PATH=/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin
```

Or use full paths:

```bash
/usr/bin/python3 /home/user/script.py
```

---

## 5) Logging & Debugging (critical 20%)

### Redirect output:

```bash
0 2 * * * /path/script.sh >> /var/log/myscript.log 2>&1
```

### Check system logs:

```bash
grep CRON /var/log/syslog
```

---

## 6) Permission Model

* User crons run as that user
* System crons can specify user:

```
* * * * * root command
```

* Control access:

  * `/etc/cron.allow`
  * `/etc/cron.deny`

---

## 7) High-Value Sysadmin Examples

### 1) Automated backups

```bash
0 3 * * * tar -czf /backup/home-$(date +\%F).tar.gz /home
```

### 2) Log cleanup (prevent disk bloat)

```bash
0 1 * * * find /var/log -type f -name "*.log" -mtime +7 -delete
```

### 3) Restart a flaky service

```bash
*/10 * * * * systemctl restart nginx
```

(Use sparingly—better to fix root cause.)

---

### 4) Monitor disk usage + alert

```bash
*/5 * * * * df -h | grep '/$' | awk '{if ($5+0 > 90) print "Disk critical"}'
```

---

### 5) Run a script at boot

```bash
@reboot /usr/local/bin/startup.sh
```

---

### 6) Database dump (production classic)

```bash
0 2 * * * mysqldump -u root -pPASSWORD dbname > /backup/db.sql
```

---

### 7) Sync files to remote server

```bash
0 * * * * rsync -az /data user@remote:/backup
```

---

### 8) Kill zombie processes (example pattern)

```bash
*/15 * * * * pkill -f bad_script.py
```

---

## 8) `/etc/cron.*` Shortcuts

Instead of writing schedules:

* `/etc/cron.daily/` → runs daily
* `/etc/cron.hourly/`
* `/etc/cron.weekly/`

Just drop executable scripts there.

---

## 9) When NOT to Use Cron

Use **systemd timers instead if you need:

* Dependency handling
* Better logging
* More precise control

---

## 10) Golden Rules

* Always use **absolute paths**
* Always **log output**
* Test commands manually before scheduling
* Escape `%` in cron (`\%`)
* Keep jobs **idempotent** (safe to run repeatedly)
